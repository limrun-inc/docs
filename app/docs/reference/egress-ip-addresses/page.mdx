# Egress IP addresses

Limrun instances send outbound traffic through shared regional egress IP addresses. If your firewall or third-party service only accepts traffic from known source IPs, allowlist every address for the Limrun region your instance runs in.

These addresses are for outbound traffic from Limrun instances to your systems. They do not describe inbound access to Limrun APIs, browser streaming URLs, or per-organization egress.

## When to use these addresses

Use this list when a customer-managed system requires source IP allowlisting before it accepts traffic from Limrun. Common examples are internal APIs, package registries, SFTP servers, database proxies, webhook receivers, and third-party services that sit behind firewall or security group rules.

## Security notes

IP allowlisting is an extra network control, not authentication. Keep API keys, mTLS, signed requests, or other application-level authentication in place for any service that accepts traffic from Limrun.

The addresses below are shared by Limrun infrastructure in each region. They are not assigned to a single customer or organization. Traffic from a region may use any address listed for that region, so allowlist the full regional set.
